# Roadmap

## Submission-critical

- Capture the instrumented `performance.log` on the physical Z6S.
- Record a public, narrated demonstration of Codex, the evidence report, the
  patch workflow, and the working LCD in no more than three minutes.
- Run the repository-local skill in a primary GPT-5.6 Codex task and save the
  event `/feedback` session ID.
- Publish the legal-only repository and complete the Devpost links.

## Next engineering iteration

- Add a defined late-presentation counter instead of implying dropped frames.
- Add a small fixture project for demonstrating RetroPort AI without requiring
  copyrighted game data or an ARM toolchain.
- Export a Markdown comparison report alongside JSON.
- Add machine-readable SPDX metadata and dependency license verification.
- Test powered USB OTG/host keyboard enumeration on the target.
- Characterize speaker impedance, volume limits, underruns, and audio latency.

## Longer term

- Generalize target contracts for other framebuffer Linux appliances.
- Add an opt-in patch-review UI without weakening the human approval gate.
- Support several source/build ecosystems through explicit adapters.
- Publish a second legal case study with freely redistributable assets.
- Track benchmark history in CI while preserving host/device provenance.

## Non-goals

- Distributing commercial game assets or vendor firmware.
- Becoming a universal emulator image or package manager.
- Automatically applying model-generated patches to hardware.
- Replacing compilers, tests, license review, or physical measurement with AI
  confidence scores.
