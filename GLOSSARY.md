# Glossary

**ABI** — The binary-level contract for calling conventions, data layout, and
object format. This target requires ARM EABI5 soft-float.

**ADB** — Android Debug Bridge. On this device it provides USB diagnostic
access; it does not prove that the same connector can host a keyboard.

**ARMv5TE** — The instruction-set level supported by the ARM926EJ-S CPU.

**Codex skill** — Repository-local instructions that define a repeatable Codex
workflow. The WOLF3D lab uses one to structure GPT-5.6 reasoning and approval.

**Evidence report** — Deterministic JSON or Markdown containing file locations,
rules, dependency pins, binary facts, and clearly labeled unknowns.

**evdev** — Linux event-device interface used for keyboard and touch input.

**fbcon** — SDL 1.2's Linux framebuffer/console backend.

**Framebuffer** — Memory scanned out to the display. The Z6S target uses
480x272 pixels at 32 bits per pixel.

**Logical resolution** — The resolution rendered by the game, 320x200 here.

**Native resolution** — The fixed physical scanout resolution, 480x272 here.

**Presentation rate** — Completed frame-presentation calls per elapsed second.
It is not automatically equal to GPU FPS or panel refresh.

**Soft-float** — An ABI that does not require hardware floating-point argument
registers. It is mandatory for this target toolchain.

**tinyalsa** — A small userspace ALSA PCM library used by the minimal mixer.

**Upstream pin** — An exact source commit used to make reconstruction
deterministic.

**WL1 / WL6** — Wolfenstein 3D data profiles for shareware and registered data.
The data files are separate copyrighted inputs and are not included.
